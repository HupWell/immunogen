# 等位基因名字太乱？先看这一页（写给非生物/非药学背景）

## 1. 你在 ImmunoGen 里要关心的只有一件事

**上游软件吐出一串 HLA 名字，下游另一个软件（如 NetMHCIIpan）要「同一种身份证的不同写法」。**  
我们的任务不是背基因，而是：**在 JSON 里用固定字段，必要时用一张「简写 → 目标工具写法」的对照表。**

可以把它想成：

- 有的人写手机号 `13800138000`  
- 有的人写 `+86 138 0013 8000`  
- **其实是同一个人**；程序跑之前需要统一成其中一种，工具才认。

---

## 2. 三个词就够（不用背）

| 词 | 你可以这样理解 |
| --- | --- |
| **HLA 分型** | 给「免疫系统用的身份证条」编号；每个人一组。 |
| **I 类 / II 类** | 两「类」条；任务里约定了 JSON 里 **I 类写哪几个键、II 类写哪几个键**（见 `hla_typing.md`）。 |
| **等位基因** | 就是「那一条上具体哪一个版本号」，常带 `*01:01` 这种**数字+冒号**，像软件版本号。 |

---

## 3. 我们的 JSON 里长什么样

- 必备：`HLA-A`、`HLA-B`、`HLA-C`（数组里一行一个等位基因）。  
- 可选：II 类如 `HLA-DRB1` 等（同样是字符串数组）。  
- 带例子请看：`data/examples/hla_typing.class_ii.example.json`。

**你只要保证：和上游 BioDriver 说好用同一套键名、同一行里不要混用两种无关格式。**

---

## 4. NetMHCIIpan 要哪种「写法」

NetMHCIIpan 是 **II 类** 结合预测里常用的一个命令行程序。它往往认这些形式的 **II 类**名字：

- 类似 **`HLA-DRB1*15:01`**（有 `HLA-` 前缀、基因名、星号、再跟数字:数字）。

你如果从别的软件里拿到的是简写，例如只有：

- `DRB1*15:01`  

**仓库里已经加了一个「默认补前缀」的小脚本**（见下节）；若你装的 NetMHCIIpan 仍报错，再到映射表里**只加你那一行**即可。

---

## 5. 两个文件、一个脚本，分工很简单

| 是什么 | 干什么 |
| --- | --- |
| **`data/hla_allele_map_netmhciipan.json`** | **手工表**：当自动规则对不上、或官方文档要求固定别名时，在 `manual_overrides` 里加一条（只有在你遇到报错、确认官方写法后改）。**示例一条（复制进 `manual_overrides` 数组里，并删掉注释）：**<br>`{"from_biodriver": "某上游写的怪字符串", "to_netmhciipan": "HLA-DRB1*15:01"}` |
| **`docs/hla_typing.md`** | **契约**：BioDriver → ImmunoGen 的 JSON 键名、OptiType/HLA-HD 和字段的对应关系。 |
| **`scripts/hla_allele_to_netmhciipan.py`** | 读 JSON + 对常见简写**自动**补 `HLA-`；命令行可试： `python scripts/hla_allele_to_netmhciipan.py DRB1*15:01` |

---

## 6. 你具体要动手的三种情况

1. **上游已经给 `HLA-DRB1*15:01` 这种**  
   通常**不用动**，等以后接好 NetMHC 再测一遍。

2. **上游只给 `DRB1*15:01`**  
   先跑脚本看自动变成什么；和 NetMHC 官方示例一致就不用管。

3. **工具报「不认识的等位基因」**  
   打开你本机 NetMHC 自带的**等位基因列表/帮助**，找到那一行的**标准写法**，把原样和正式写法各填进 `manual_overrides` 里一行。  
   **只改有问题的，不要全表重写。**

---

## 7. YAML 可以吗？

可以。用 **同样字段** 做成 YAML 也行；当前仓库为简单起见用 **一个 JSON 文件** 作唯一来源。若你团队更习惯 YAML，保持 `manual_overrides` 结构一致即可在 CI 里互转，不必再维护两套不同字段。

---

*能跑通、能对齐官方示例即可；有不确定以「你本机安装的工具 + 它的帮助文档」为准。*
