## 一、一句话结论

## 五、全流程各阶段的「模型 / 工具分工」（老板可读版）

| 阶段 | 模型 / 工具 | 对业务的意义 |
|------|---------------|----------------|
| 候选输入 | BioDriver | 患者突变肽、VAF、HLA 分型等上游输入 |
| MHC-I 结合 | NetMHCpan | 评估 I 类 HLA 呈递 / 亲和力相关信号 |
| MHC-I 增强（可选） | BigMHC | 第二路交叉验证，降低单模型偏见 |
| MHC-II 结合 | NetMHCIIpan | 覆盖 II 类（CD4 相关）呈递维度 |
| 免疫原性 | DeepImmuno / PRIME / Repitope | 多维免疫原性相关评分，辅助排序 |
| 结构建模 | PANDORA + MODELLER | 产出 peptide–MHC 三维结构交付 SimHub |
| 结构复核（可选） | ColabFold / AFM | 对重点候选独立复核（本轮 AFM 置信度已提升至 pLDDT≈83+），成本高 |
| mRNA 序列优化 | LinearDesign（当前主线） | 密码子优化，形成可合成的 mRNA 方案 |
| RNA 稳定性 | RNAfold / RNAeval / RNAplfold | RNA 二级结构与局部稳定性风险提示 |
| 下游动力学 | SimHub MD | 用模拟验证复合物稳定性等行为 |

## 六、多条 mRNA 的「性价比」决策模板（推荐给老板的拍板逻辑）

原则：**先低成本广撒网，再高成本精验证**。

| 层级 | 手段 | 成本 | 产出 |
|------|------|------|------|
| L1 | 多模型打分 + 一致性筛选 | 低 | 淘汰明显短板候选，缩小 TOP 清单 |
| L2 | 结构（PANDORA）+ RNA 稳定性 | 中 | 得到「看起来像能交货」的组合 |
| L3 | 可选 AFM/ColabFold 复核 | 中高 | 只对少数赢家加一道保险 |
| L4 | SimHub MD | 高 | 只对 Finalist 开火，避免浪费预算 |

可参考的「并行方案」命名（便于会议对齐）：

| 方案代号 | 策略侧重 | 成本感 | 说明 |
|---------|----------|--------|------|
| A 稳妥版 | 现有 TOP 肽 + LinearDesign | 低 | 已适合当前阶段验收叙事 |
| B 免疫优先版 | 加权免疫原性模型 | 低～中 | 更激进追求免疫信号，可能要牺牲一部分稳定性权衡 |
| C 结构优先版 | PANDORA 后再 AFM/ColabFold 复核 | 中～高 | 进 SimHub 前「结构不过关就砍掉」 |
| D 稳定性优先版 | 同肽多套密码子 RNA 中选优 | 中 | mRNA「可制造性」友好 |
| E 成本优先版 | 多模型一致性高分才送 MD | 低 | SimHub 次数少，覆盖面也小 |
